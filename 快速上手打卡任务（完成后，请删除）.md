## 相关建议
- **请勿修改文件夹结构**：仓库中预设的文件夹结构（ {icon=folder} Assistants、 {icon=folder} Books、 {icon=folder} Connections、 {icon=folder} Documents、 {icon=folder} Extras文件夹、{icon=folder-tree}子文件的名称、文件名称和路径）请不要修改，但是你*可以自己增加文件夹*，具体[[1-预制仓库移植使用#哪些文件能修改（❗❗❗重要，牢记❗❗❗）|点击这里]]了解
- **记住自己修改了哪些内容**：仓库中预设的软件设置、插件设置也不建议直接修改，若要自定义，请**记住自己进行了哪些设置‼️‼️**，以便出现问题时还原。（很多用户自己自定义、或修改了设置，忘记了做了什么调整，*导致系统不能正常运行*，联系我找问题，这个过程很费时间，且不一定能还原，所以请记住自己进行了哪些设置）
- **请保留下载的「预制仓库」压缩包**：如果文件损坏或被修改无法还原，可以从压缩包中找到原有的文件或文件夹恢复
- **多端同步**：请自行搭建（仓库没有限制任何同步方式），多端同步可能会导致设置文件丢失或损坏，由此产生的问题非常非常多，所以请谨慎使用多端同步（如遇到问题，第一时间想想是不是同步造成的），关于同步的教程，你可以[[19-关于同步|详见这里]]  
- **导入你的笔记**：我们先不着急导入你自己的已有仓库笔记，看完下面的教程、完成任务、对模块化OB预制仓库有一定了解后，在文末查看如何{icon=import} [[快速上手打卡任务（完成后，请删除）#导入你的笔记|导入笔记]] 也不迟

1. **方式1：通过快速切换命令创建笔记(快捷键Ctrl + K)**
	1. 步骤：快捷键<kbd style="background: var(--background-primary); border: 1px solid var(--text-muted); border-radius: 3px; padding: 1px 6px; margin: 5px;">Ctrl + K</kbd>（或左侧功能区 {icon=file-search} 按钮）→输入文件名→<kbd style="background: var(--background-primary); border: 1px solid var(--text-muted); border-radius: 3px; padding: 1px 6px; margin: 5px;">Shift + Enter</kbd>  
	2. 技巧：输入的文件名如果仓库中有同名笔记，则直接打开原有同名笔记进行编辑，如果没有同名笔记，<kbd style="background: var(--background-primary); border: 1px solid var(--text-muted); border-radius: 3px; padding: 1px 6px; margin: 5px;">Shift + Enter</kbd>可创建一篇新的笔记，此方式如果运用得当，是最推荐的笔记创建方式
	3. 注意：输入的文件名，如果仓库有完全相同的笔记，即使使用<kbd style="background: var(--background-primary); border: 1px solid var(--text-muted); border-radius: 3px; padding: 1px 6px; margin: 5px;">Shift + Enter</kbd>，也只会打开原有笔记，如果不希望打开原有笔记，请重新输入新的笔记名称（与原有笔记进行区别）
	4. 提示：“查看近期文件”选项，实际上与左侧功能区 {icon=file-search} 按钮是同一个功能，在不打开标签页时，通过该按钮创建笔记，会更加快捷。
2. **方式2：通过右侧边栏按钮创建笔记（快捷键Alt + →）**
	1. 步骤：<kbd style="background: var(--background-primary); border: 1px solid var(--text-muted); border-radius: 3px; padding: 1px 6px; margin: 5px;">Alt+ →</kbd>打开右侧边栏 → 点击按钮区域（蓝色字体的按钮）→ **选择**笔记存放路径 → 输入笔记名称 → 确定，笔记将自动归集到指定路径中。
	2. 提示：如果右侧边栏找不到按钮，请点击左侧功能区 {icon=panels-top-left} 图标 → 切换到“笔记工作区”。
	3. 注意：模块化OB预设的领域（工作、学习、生活），具有普遍适用性（也最稳定），**非必要不建议创建自己的自定义领域**。若确实需要创建自己的专属领域，请在Content Protection插件设置中进行创建。
3.  **方式3：通过标题栏右上角的{icon=plus}按钮创建笔记（快捷键Ctrl + N）**

%%说明：
1. **使用内置网页浏览器**
	1. 步骤：在输入框输入搜索内容 → 点选合适的搜索引擎
	2. 技巧：在obsidian内部打开的网页内容，可以通过右上角的三个点「更多选项」，选择「Save to vault」，以保存网页内容（保存内容或不完整是正常现象）

2. **嵌入常用模板（快捷键Alt(option)+Q）**
	1. 注意：使用此按钮需要将`模板`放到`Assistants>Templater>片段`文件夹中。
	2. 步骤：将光标放在笔记嵌入位置 → 使用快捷键<kbd style="background: var(--background-primary); border: 1px solid var(--text-muted); border-radius: 3px; padding: 1px 6px; margin: 5px;">Alt + Q</kbd>（**视频中的“嵌入模板按钮”已经弃用**）→ 搜索或选择需要添加的模板（注意：视频中的选项与本仓库不一致，请自行按照下面的方法自定义需要的模板）
	3. 技巧：
		1. 此按钮嵌入的模板格式没有任何限制，只需要将你的模板放在“{icon=folder}片段”文件夹中，就可以通过快捷键<kbd style="background: var(--background-primary); border: 1px solid var(--text-muted); border-radius: 3px; padding: 1px 6px; margin: 5px;">Alt + Q</kbd>嵌入“{icon=folder}片段” 文件夹的模板。
		2. 通过此方式，你可以将常用的模板、模块放在“{icon=folder}片段” 文件夹中，方便调用。
3. **给笔记设定*笔记类型、详述计划、笔记样式、文档标签***
	1. 步骤：点击笔记头部的卡片盒笔记区域 → 选择相应的功能
	2. 技巧：输入标签时，现已支持自动补全（如果需要关闭此功能，请在content protection插件设置中关闭）
4. **学会「创建主题」并「选择主题」关联笔记**
	1. 步骤：点击「创建主题」按钮 → 选择索引卡保存路径（*注意不是输入*）→ 输入主题索引卡名称（注意不要换行或使用特殊符号）→ 确定 → 点击「选择主题」按钮 → 在弹出的列表中选择已经创建的索引卡，[{icon=play-circle}点击此链接播放视频](https://ytmaps.vip/paly/guide/canvas%E8%87%AA%E5%8A%A8%E5%9B%9E%E9%93%BE.html)   
	2. 技巧：
		1. 「选择主题」后，将自动在canvas白板中嵌入当前笔记，无需再手动添加到canvas白板中。这意味着你可以将具有共同主题的笔记平铺在白板中，进行思维导图、整理、编辑、二次关联等操作。
		2. 若在白板中误删了已经嵌入的笔记，不好恢复时，最后的解决途径是：删除所有已经嵌入的笔记，然后使用快捷键<kbd style="background: var(--background-primary); border: 1px solid var(--text-muted); border-radius: 3px; padding: 1px 6px; margin: 5px;">Alt+Shift+B</kbd>（或右侧工具栏 {icon=link} 图标）一键批量嵌入所有反向链接笔记
		3. [[文档数据库-Bases.base|此文档数据库]]可快速**批量设置**笔记主题、类型、标签等
		4. 选择主题后，如果需要**删除已经选择的主题**，请在右侧边栏“{icon=info}笔记属性”中删除。
5. **学会移动笔记**
	1. 步骤：在笔记头部的“卡片盒笔记”展开「移动文档」按钮→点击对应的按钮
	2. 技巧：如果点击“其他文件夹”按钮时，会出现<font color="#ff0000">`[MB_UNLOADED] Embed`</font>提示，这是可预见的问题（暂时无法从根本上解决）。或者修改笔记名称，也会出现类似问，可通过快捷键<kbd style="background: var(--background-primary); border: 1px solid var(--text-muted); border-radius: 3px; padding: 1px 6px; margin: 5px;">Alt + R</kbd>刷新解决
	3. 提示：当你的笔记数量较少时，移动笔记按钮看起来没什么大的作用，但是当你的笔记数量越来越庞大（移动文档将变成难题），移动笔记按钮将发挥重要作用。
6. **学会记录跟踪笔记进度**
	1. 步骤：点击进度条 → 设定value值（0～100，建议10的倍数）→ 快捷键<kbd style="background: var(--background-primary); border: 1px solid var(--text-muted); border-radius: 3px; padding: 1px 6px; margin: 5px;">Alt + X</kbd>，点击右上角弹出的通知 → 查看「今日/未来1周/逾期」阐述的笔记
	2. 技巧：value值设定为100时，[[今日需处理的笔记]]不再显示该笔记
	3. 注意：新版本已支持直接点击进度条设定进度

	1. 传统的笔记管理方式：通常先建立固定的 {icon=folder-tree} 文件夹结构，再将笔记逐一放入对应位置，这种方式虽然条理清晰，却也在无形中限制了思维的延展，仿佛过早地为思想设定了路径，难以激发更多关联与可能性。
	2. 模块化Obsidian理念：借助{icon=link-2}双向链接、{icon=tags}标签与{icon=database}数据库等功能，释放了笔记的组织潜力，你可以先专注记录，再通过链接与笔记主题动态关联，在合适的时机灵活整理，让想法自由碰撞，孕育出更多新的洞察与创意。

	3. 若不需要此功能，或者暂时不用，请在`设置 → 第三方插件`关闭Douban插件。

# 导入笔记

### 导入你的笔记
[{icon=play-circle}点击此链接播放视频](https://ytmaps.vip/paly/guide/%E7%AC%94%E8%AE%B0%E5%AF%BC%E5%85%A53.html)
- [ ] 通过Note Importer插件导入你的笔记⭐⭐⭐⭐
%%说明：
1. 注意：
	1. 目前插件只导入笔记和附件，不包含插件、主题、css样式代码等内容
	2. 若导入的笔记需要其他插件支持，请自行从自己的已有仓库中拷贝到模块化OB预制仓库({icon=folder} .obsidian>plugins文件夹中）
	3. 目前导入的附件支持格式：png, jpg, jpeg, gif, bmp, svg, webp, pdf, mp4, mp3, wav, ogg, webm, mov, base, canvas, avif, flac, m4a, 3gp, mkv, ogv, doc, docx, xls, xlsx, zip, rar, 7z, txt, ppt, pptx
	4. 笔记导入的位置为：{icon=folder} Documents/I.P.A.R.A/导入笔记
	5. 附件导入的位置为：{icon=folder} Extras/导入
2. 技巧：
	1. 导入笔记后，您可以在这个[[文档数据库-Bases.base]]文档中快速分类、整理笔记
	2. 但需要注意，通过[[文档数据库-Bases.base]]文档添加的“卡片盒笔记主题”无法主动将笔记添加到对应的canvas白板，你可以后续点击右侧工具栏 {icon=link} 图标将笔记嵌入到白板中。
3. 建议：
	1. {icon=fish-symbol}鱼先生建议大家在导入笔记、整理笔记时，不要一味图快（否则笔记容易继续吃灰），慢下来，好好回顾、分类整理已有的笔记，重拾这些笔记的潜在价值，或许比起快速整理更加重要
